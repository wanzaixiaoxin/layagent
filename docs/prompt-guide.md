# 提示词编写指南

## 如何写好游戏描述

游戏描述的质量直接影响生成结果。遵循以下建议：

### 包含关键信息

一个好的游戏描述应该包含：

1. **游戏类型** — 横版闯关、射击、益智...
2. **核心玩法** — 玩家做什么？怎么操作？
3. **视觉风格** — 卡通、写实、像素...
4. **角色/物品设定** — 主角外观、道具效果
5. **目标平台** — H5、微信小游戏...

### 示例

**一般描述：**
> 一个跑酷游戏

**优秀描述：**
> 一个忍者猫咪横版跑酷游戏，玩家控制一只穿黑色忍者服的橘猫在森林中自动奔跑，点击屏幕跳跃，空中可以二段跳。要躲避木箱和陷阱，收集金币和加速道具。卡通Q版风格，色彩鲜艳，适合手机游玩。

## 风格一致性技巧

### 1. 风格锚点

在描述中明确指出参考风格：

> "类似《猫咪大战争》的简洁可爱风格"
> "类似《蔚蓝》的像素艺术风格"
> "类似《纪念碑谷》的几何极简风格"

### 2. 色彩指定

提供具体的色彩偏好：

> "主色调为绿色和棕色，森林主题"
> "霓虹配色，赛博朋克风格"

### 3. 生成后微调

生成提示词包后，可以手动调整风格圣经中的规则，确保所有提示词的一致性。

## 与 AI 绘画工具的配合

### Midjourney

- 使用提示词包中的 `basePrompt` 直接作为 prompt
- 在末尾添加 `--style raw` 获得更干净的游戏素材
- 使用 `--niji 6` 获得动漫风格角色
- 使用 `--ar 1:1` 确保方形素材

### Stable Diffusion

- 将 `basePrompt` 放入正向提示词
- 将 `negativePrompt` 放入负向提示词
- 使用 ControlNet 控制角色姿态一致性
- 推荐模型：AnythingV5、Counterfeit-V3.0

### 推荐工作流

1. 在 LayaGen 中生成提示词包（Markdown 格式）
2. 逐个复制提示词到 AI 绘画工具
3. 第一张生成的图片作为后续的风格参考
4. 统一使用相同的种子值或参考图
5. 生成完成后放入 LayaAir 项目的 res/ 目录
